לא יודע לתכנת? מושלם — זה בדיוק בשבילך
Claude Code מאפשר לאנשים בלי רקע בקוד לבנות אפליקציות אמיתיות. אבל יש כמה מושגים טכניים שחוזרים על עצמם. הפרק הזה מסביר אותם בפשטות — מלמעלה עד למטה. תקרא אותו פעם אחת ותבין הכל.
🖥️
מה זה טרמינל / שורת פקודה?
חלון טקסט שבו כותבים פקודות ישירות למחשב — במקום ללחוץ על כפתורים ועכבר. ב-Windows קוראים לזה Command Prompt או PowerShell. ב-Mac קוראים לזה Terminal. Claude Code עובד בתוך הטרמינל הזה.
🟢
מה זה Node.js?
תוכנה שמאפשרת להריץ קוד JavaScript מחוץ לדפדפן — ישירות על המחשב שלך. Claude Code בנוי על Node.js, לכן צריך להתקין אותו קודם. זה כמו "מנוע" שבלעדיו הכלי לא יכול לעבוד.
📦
מה זה npm?
מנהל החבילות של Node.js. "חבילה" = תוכנה מוכנה שמישהו כתב ואתה יכול להשתמש בה. npm install = "תוריד והתקן את התוכנה הזו". npm מגיע אוטומטית עם Node.js — לא צריך להתקין בנפרד.
🔧
מה זה "התקנה גלובלית" (-g)?
כשמתקינים תוכנה עם -g (global), היא זמינה מכל תיקייה במחשב. בלי -g, התוכנה עובדת רק בתיקייה שבה התקנת אותה. Claude Code צריך התקנה גלובלית כי רוצים להשתמש בו בכל פרויקט.
🌿
מה זה Git?
מערכת לגיבוי וניהול גרסאות של קוד. כל פעם שעושים commit, Git שומר "צילום" של הקוד ברגע הזה. אפשר תמיד לחזור לגרסה ישנה. זה כמו Ctrl+Z אבל לאינסוף שלבים לאחור לנצח.
🐙
מה זה GitHub?
אתר שמאחסן את הקוד שלך ב-"ענן" (שרתים של GitHub). Git = הכלי על המחשב שלך. GitHub = המקום באינטרנט שבו שומרים את הגיבויים. git push = שליחת השינויים מהמחשב ל-GitHub.
🚀
מה זה Vercel?
שירות שמעלה את האתר שלך לאינטרנט תוך שניות — בחינם. אתה מחבר אותו ל-GitHub, ובכל פעם שמעדכנים את הקוד ב-GitHub, Vercel מעדכן את האתר באוטומט. אין צורך בשרת, אין הגדרות מסובכות.
🗂️
מה זה תיקיית פרויקט?
תיקייה רגילה על המחשב שמכילה את כל קבצי הפרויקט. כשפותחים טרמינל ו"נכנסים" לתיקייה (cd שם-תיקייה), כל הפקודות שמריצים משפיעות על הקבצים בתיקייה הזו בלבד.
🔑
מה זה API?
ממשק שמאפשר לתוכנה אחת "לדבר" עם תוכנה אחרת. Claude Code משתמש ב-API של Anthropic כדי לשלוח את ההוראות שלך ל-Claude ולקבל תשובות. API Key = מפתח זיהוי אישי שמאשר שאתה הלקוח.
📝
מה זה קובץ .md (Markdown)?
קובץ טקסט פשוט עם עיצוב בסיסי. # כותרת = כותרת גדולה. **מודגש** = טקסט מודגש. - פריט = רשימה. Claude קורא קבצי .md ומבין את המבנה שלהם — לכן CLAUDE.md כתוב בפורמט הזה.
🌐
מה זה TypeScript?
גרסה מתקדמת של JavaScript (שפת התכנות של הדפדפן). TypeScript מוסיף "כללי בטיחות" שעוזרים למצוא שגיאות לפני שהקוד רץ. Claude Code מציע להשתמש בו בפרויקטים רציניים — אבל לא חייבים בהתחלה.
⚛️
מה זה React / Next.js?
React = ספריית JavaScript לבניית ממשקי משתמש (כפתורים, טפסים, רשימות). Next.js = מסגרת שבנויה על React ומוסיפה לה יכולות כמו ניהול דפים, קישורים, ו-backend בסיסי. יחד הם הבסיס לרוב האפליקציות האינטרנטיות.
🐘
מה זה Supabase?
שירות בסיס נתונים בענן — המקום שבו שומרים את המידע של האפליקציה (משתמשים, הזמנות, כל דבר). מגיע עם מערכת הרשמה/כניסה מוכנה (Auth), אבטחה (RLS), ו-API אוטומטי. בחינם עד לנפח מסויים.
🏗️
מה זה "ארכיטקטורה"?
האופן שבו בנוי הפרויקט — אילו קבצים יש, איך הם מחוברים, מה אחראי על מה. כמו תוכנית הבנייה של בית לפני שמתחילים לבנות. Claude Code עובד הכי טוב כשיש ארכיטקטורה ברורה מראש.
🐛
מה זה Bug / שגיאה?
קוד שכתוב אבל לא עובד כמו שצריך. כשיש שגיאה, הטרמינל או הדפדפן מראים הודעת שגיאה (Error). הפתרון: להעתיק את הודעת השגיאה, להדביק אותה ל-Claude, ולכתוב "fix". ברוב המקרים הוא מתקן לבד.
🧪
מה זה Tests?
קוד שבודק שקוד אחר עובד נכון. כמו מבחן שהקוד עובר לפני שמעלים אותו. npm test = הרצת כל הבדיקות. Claude יוצר tests אוטומטית — חשוב לוודא שהם עוברים לפני כל commit.
✅
לא חייבים לזכור הכל!
כשנתקל במושג לא מוכר במהלך הקריאה — חזור לפרק הזה. עם הזמן הכל הופך לטבעי. Claude Code עצמו יכול להסביר כל מושג אם תשאל אותו: "מה זה X ולמה צריך אותו?"

מה זה Claude Code?

Claude Code הוא CLI agent — כלי שורת פקודה שמריץ Claude ישירות על הסביבה המקומית שלך. הוא יכול לקרוא קבצים, לכתוב קוד, להריץ פקודות, ולנהל את הפרויקט שלך מקצה לקצה — בלי שתצטרך להעתיק-הדביק שום דבר.

אתה פשוט פותח טרמינל בתיקיית הפרויקט, מקליד claude, ומתחיל לתת הוראות בשפה טבעית.

🌐 claude.ai (הצ'אט)
  • עובד בדפדפן
  • מושלם לשאלות ורעיונות
  • לא ניגש לקבצים שלך
  • צריך להדביק קוד ידנית
  • אין גישה לטרמינל
⚡ Claude Code (CLI)
  • עובד מהטרמינל
  • ניגש לכל קבצי הפרויקט
  • מריץ פקודות shell
  • מבצע שינויים ישירות
  • מנהל git, tests, deploys
💡
מתי להשתמש ב-Claude Code?
כשאתה עובד על פרויקט קוד אמיתי — לא לשאלות תיאורטיות. Claude Code מתאים לפיתוח, debugging, refactoring, כתיבת tests, ואוטומציה של workflows.

פרק 2

התקנה והגדרה

💡
מה הולך לקרות עכשיו?
נעבור 4 שלבים פשוטים: נתקין Node.js → נתקין Claude Code → נתחבר לחשבון → נפתח את הכלי. כל שלב לוקח 2-3 דקות. עושים פעם אחת ואחר כך לא צריך לחזור.
0
התקן Node.js דרישה מוקדמת
מה זה Node.js? תוכנה שמריצה JavaScript על המחשב שלך. Claude Code בנוי עליה ולא יכול לעבוד בלעדיה — כמו שמכונית לא יכולה לנסוע בלי מנוע.

כיצד לבדוק אם כבר מותקן: פתח טרמינל והקלד:
bash
node --version
אם מופיע מספר כמו v20.11.0 — מעולה, Node כבר מותקן. צריך שיהיה גרסה 18 ומעלה.
אם מופיע שגיאה — כנס לאתר nodejs.org, הורד את הגרסה LTS (הירוקה) והתקן כרגיל.
1
התקן את Claude Code npm
מה קורה כאן? הפקודה npm install אומרת "הורד והתקן את התוכנה הזו". ה--g אומר "גלובלית" — זמינה בכל מקום במחשב. @anthropic-ai/claude-code הוא שם הכלי הרשמי.

פתח טרמינל והרץ:
bash
npm install -g @anthropic-ai/claude-code
ההתקנה לוקחת 30-60 שניות. תראה הרבה טקסט שרץ — זה נורמלי. בסוף צריך לקבל הודעה שאומרת שההתקנה הצליחה, ואפשר לבדוק עם claude --version.
2
התחבר לחשבון Anthropic auth
למה צריך חשבון? Claude Code הוא שירות בתשלום (יש תוכנית חינמית מוגבלת). הפקודה claude auth פותחת דפדפן שמבקש ממך להתחבר לחשבון claude.ai שלך — זה אותו חשבון שיש לך לצ'אט.

אם אין לך חשבון — פתח אחד קודם ב-claude.ai, זה חינמי.
bash
claude auth
הדפדפן נפתח → התחבר → לחץ "Authorize" → חזור לטרמינל. תראה הודעת אישור שאומרת שההתחברות הצליחה.
3
נווט לתיקיית הפרויקט שלך cd
מה זה cd? קיצור של "Change Directory" — כמו לפתוח תיקייה בסייר הקבצים, רק דרך הטרמינל. חשוב מאוד: תמיד להפעיל Claude Code מתוך תיקיית הפרויקט שלך — הוא עובד על הקבצים שנמצאים שם.
bash
# דוגמה - החלף את הנתיב בנתיב האמיתי שלך
cd C:\Users\שמך\Documents\my-project    # Windows
cd /Users/שמך/Documents/my-project     # Mac/Linux
💡
טיפ מהיר — גרור תיקייה לטרמינל
ב-Windows/Mac: כתוב cd (עם רווח) ואז גרור את התיקייה לתוך חלון הטרמינל — הנתיב יוקלד אוטומטית. הכי פשוט!
4
הפעל את Claude Code 🚀
עכשיו שאתה בתיקיית הפרויקט — פשוט הקלד:
bash
claude
Claude Code יסרוק את כל קבצי הפרויקט, יבין את המבנה, ויציג prompt שבו אתה יכול לכתוב בשפה טבעית מה אתה רוצה.

מה כותבים? כל דבר! "תסביר לי מה יש פה", "תוסיף כפתור ירוק בעמוד הבית", "יש שגיאה בקובץ X — תתקן". Claude מבין עברית ואנגלית.
⚡
קיצור דרך מתקדם: alias cc
מה זה alias? קיצור לפקודה ארוכה. במקום לכתוב claude --dangerously-skip-permissions בכל פעם, יוצרים alias שמאפשר לכתוב רק cc.

הדגל --dangerously-skip-permissions אומר לClaude "אל תבקש אישור על כל פעולה קטנה" — מאיץ מאוד את העבודה בסביבת פיתוח פרטית.

להוסיף לקובץ ההגדרות של הטרמינל (~/.zshrc ב-Mac, ~/.bashrc ב-Linux):
alias cc="claude --dangerously-skip-permissions"

פרק 3

קובץ CLAUDE.md — הכלי הכי חשוב

מה זה CLAUDE.md בכלל? קובץ טקסט רגיל (בפורמט Markdown) שיוצרים בתיקיית הפרויקט בשם CLAUDE.md. הוא לא קובץ קוד — זה קובץ הוראות בשפה אנושית. Claude קורא אותו אוטומטית בתחילת כל שיחה, לפני שמתחילים לעבוד.

דמיין שאתה שוכר מפתח חדש לפרויקט שלך. הוא לא יודע כלום על הפרויקט — הוא לא יודע שכתבת React ולא Angular, שמשתמשים ב-pnpm ולא ב-npm, שאסור לשנות קבצי ה-auth. כל זה צריך להסביר לו מחדש בכל פגישה. CLAUDE.md = מסמך הנחיות שהמפתח החדש (Claude) קורא לפני כל פגישה — ולא שוכח.

⚠️
שים לב
CLAUDE.md הוא ה-הבדל הכי גדול בין תוצאות מאכזבות לתוצאות מדהימות. מפתחים שמדלגים על זה מתלוננים ש-Claude "לא מבין" — בדרך כלל הבעיה היא שאין CLAUDE.md.

מה לכתוב ב-CLAUDE.md?

  • 🌐 שפה: "תענה תמיד בעברית" / "Answer in English"
  • 🏗️ ארכיטקטורה: תיאור המבנה הכללי של הפרויקט
  • 📏 סגנון קוד: ESLint, Prettier, naming conventions
  • 🚫 מה אסור לגעת: קבצים/תיקיות שאסור לשנות
  • 🔑 APIs ושירותים: אילו שירותים בשימוש ואיך
  • 🧪 Testing: איך כותבים tests בפרויקט
  • 📦 Package manager: npm / yarn / pnpm
CLAUDE.md — דוגמה מלאה 
# הנחיות לעבודה — פרויקט MyApp

## שפה ותקשורת
- תענה תמיד בעברית
- הסברים טכניים — בעברית אבל שמות קבצים/פקודות באנגלית

## ארכיטקטורה
- Frontend: React 18 + TypeScript + Vite
- Backend: Node.js + Express + PostgreSQL
- State: Zustand (לא Redux!)
- Styling: Tailwind CSS בלבד — אסור CSS modules

## כללי עבודה
- לפני כל שינוי גדול — לשאול ולקבל אישור
- תמיד לכתוב TypeScript strict mode
- לא לשנות שום קובץ ב-/db/migrations/ ישירות
- package manager: pnpm בלבד (לא npm!)

## מה אסור לגעת
- /src/core/auth/ — מנוהל על ידי צוות אחר
- .env* — לא לשנות, לא ליצור
- /public/legacy/ — קוד ישן שלא נוגעים בו

## Testing
- כל פיצ'ר חדש חייב unit test ב-Vitest
- Integration tests ב-/tests/integration/
- לא מדביקים ב-main בלי tests ירוקים

## Naming conventions
- קבצי components: PascalCase.tsx
- hooks: use + camelCase.ts
- utils: camelCase.ts
- קבצי API routes: kebab-case.ts

פרק 4

תהליך העבודה הנכון

למה צריך תהליך בכלל? כשעובדים עם Claude Code בלי תהליך — כמו שנוסעים בלי מפה. מגיעים לאי שם, לא בטוח לאן. Claude יתחיל לכתוב קוד מיד, יכתוב משהו שלא בדיוק רצית, תבקש לשנות, הוא ישנה משהו שישבור משהו אחר... מעגל מתסכל.

התהליך הנכון שמקצוענים משתמשים בו: קודם מחקר, אחר כך תכנון, אחר כך ביצוע. לא לדלג על שלבים. לא "תכתוב לי את כל המערכת" — אלא שלב אחרי שלב.

א
מחקר (Research) לפני הכל
בקש מ-Claude לחקור לעומק לפני שמתחילים לכתוב קוד. השתמש במילים כמו "deeply", "in great detail", "go through everything". בקש שיכתוב research.md עם הממצאים.
💬
דוגמת prompt
"Deeply investigate the codebase and understand how authentication works. Go through every relevant file in great detail. Write your findings to research.md — don't write any code yet."
ב
תכנון (Plan) קריטי
בקש מ-Claude לכתוב תוכנית יישום מפורטת לקובץ plan.md — מבלי לכתוב קוד. עבור לבעיות מורכבות: לחץ Shift+Tab כדי להיכנס ל-Plan Mode. הוסף "think hard" או "think deeply" להוראות מורכבות.
prompt לתכנון 
כתוב תוכנית יישום מפורטת ל-plan.md.
חשוב hard על כל edge case.
אל תכתוב קוד עדיין — רק תוכנית.
ג
סקירה ואנוטציות
פתח את plan.md בעורך הקוד שלך. קרא אותו בעצמך והוסף הערות inline בכל מקום שיש לך שאלה או הסתייגות. שלח את Claude בחזרה לעדכן את התוכנית לפי ההערות.
✅
המלצה
זה השלב שבו המומחיות שלך נכנסת לתמונה. Claude טוב בקוד — אתה יודע את הדרישות העסקיות. השקע כאן.
ד

פרק 5

פקודות מובנות חשובות

מה זה "פקודות מובנות"? בזמן שיחה עם Claude Code, יש פקודות מיוחדות שמתחילות עם / (סלש). הן לא הוראות ל-Claude — הן הוראות לכלי עצמו. למשל /clear מוחקת את זיכרון השיחה, /rewind מחזירה למצב קודם.

בנוסף יש קיצורי מקלדת שעובדים בתוך Claude Code. המועילים ביותר: Esc לעצירת Claude באמצע, ו-Shift+Tab למצב תכנון בלבד.

פקודה / קיצור תיאור מתי להשתמש
/help רשימת כל הפקודות הזמינות כשאתה רוצה לגלות יכולות חדשות
/status מצב ה-session הנוכחי — כמה tokens נצרכו לפני compact, לניטור עלויות
/clear מחיקת כל ההיסטוריה של ה-session כשמתחילים משימה חדשה לגמרי
/compact דחיסת ההיסטוריה לסיכום קצר כשה-context מתמלא ורוצים להמשיך
/rewind חזרה ל-checkpoint קודם כשמשהו השתבש ורוצים לחזור אחורה
/resume פתיחת session picker לחזור לשיחה קודמת שנשמרה
/rename שינוי שם ל-session הנוכחי לארגון sessions לפי נושאים
/branch פיצול ה-session הנוכחי לניסוי approach שני במקביל
Esc עצירת Claude באמצע ביצוע כשרוצים לתקן הוראה שגויה
Esc + Esc פתיחת תפריט checkpoints לניווט בין נקודות שמירה
Shift+Tab כניסה / יציאה מ-Plan Mode לתכנון ללא ביצוע

פרק 6

Custom Slash Commands

Custom Slash Commands מאפשרים לך ליצור פקודות מותאמות אישית לפרויקט שלך. יוצרים קובץ Markdown בתיקייה .claude/commands/ — ואפשר להפעיל אותו עם /שם-הפקודה מתוך Claude.

כל קובץ .md בתיקייה הזו הופך אוטומטית לפקודה זמינה.

.claude/commands/new-component.md 
# /new-component

צור React component חדש בשם $ARGUMENTS.

## דרישות
1. קובץ TypeScript (.tsx) ב-/src/components/
2. Interface מלא ל-Props
3. Storybook story (.stories.tsx) לצד הקובץ
4. Unit test בסיסי עם Vitest
5. Export מ-/src/components/index.ts

## סגנון
- Functional component בלבד (לא class)
- CSS דרך Tailwind utility classes
- memo() אם ה-component נטהר (pure)
- aria-label לאלמנטים אינטראקטיביים

## דוגמה לשימוש
/new-component UserAvatar
.claude/commands/review-security.md 
# /review-security

בצע סקירת אבטחה מקיפה על הקוד שצוין: $ARGUMENTS

## בדוק את הנושאים הבאים
- SQL injection ב-queries
- XSS vulnerabilities ב-rendering
- CSRF protection ב-forms
- Authentication bypasses
- Sensitive data exposure (passwords, keys)
- Insecure direct object references
- Rate limiting חסר

## פורמט הדוח
עבור כל ממצא:
- **חומרה:** CRITICAL / HIGH / MEDIUM / LOW
- **קובץ + שורה**
- **תיאור הבעיה**
- **קוד לתיקון**

צור תוכנית refactoring מפורטת עבור: $ARGUMENTS

## תהליך

2. זהה code smells וטכנות חוב
3. הצע שלבי refactoring מסודרים
4. שמור backward compatibility

## מה לכסות
- מה משתנה ולמה
- סיכוני regression

פרק 7

Hooks — אוטומציה על אירועים

מה זה Hooks בכלל? דמיין שיש לך עוזר שעובד בשבילך — כל פעם שהוא שומר קובץ, אתה רוצה שיריץ בדיקה אוטומטית. כל פעם שהוא מסיים משימה, תישלח לך הודעה. Hooks = הוראות אוטומטיות שמופעלות כשקורה משהו.

מגדירים Hooks בקובץ .claude/settings.json בתיקיית הפרויקט. הם עוצמתיים מאוד — אבל לא הכרחיים בהתחלה.

4 סוגי אירועים:

  • PreToolUse — לפני שClaude עושה משהו (למשל: לפני שכותב קובץ). שימושי ל: "אל תעשה X בלי לשאול אותי"
  • PostToolUse — אחרי שClaude עשה משהו (למשל: אחרי שמירת קובץ). שימושי ל: "אחרי כל שמירה הרץ בדיקה"
  • Stop — כשClaude מסיים תגובה. שימושי ל: "שלח לי התראה שהוא סיים"
  • SessionStart — בתחילת שיחה חדשה. שימושי ל: "בתחילת כל שיחה טען את המידע העדכני"
.claude/settings.json — Hook: Linter אחרי שמירה 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --fix 2>&1 | tail -5"
          }
        ]
      }
    ]
  }
}
.claude/settings.json — Hook: מניעת commit בלי tests 
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -q 'git commit'; then npm test --passWithNoTests || (echo 'ERROR: Tests must pass before commit!' && exit 1); fi"
          }
        ]
      }
    ]
  }
}
.claude/settings.json — Hook: הודעה כשClaude מסיים 
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude סיים את המשימה\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
⚠️
שים לב לאבטחה
Hooks מריצים פקודות shell עם הרשאות מלאות. וודא שהפקודות בטוחות ולא חשופות לCommand Injection אם הן מקבלות קלט חיצוני.

פרק 8

Subagents ו-MCP Servers

מה זה Subagent? דמיין שיש לך משימה גדולה: לבדוק אבטחה, לכתוב tests ולעצב UI — שלוש עבודות שונות. במקום שClaude יעשה אחת אחרי השנייה (איטי), אפשר לפתוח 3 חלונות טרמינל ולהפעיל Claude Code בכולם בו-זמנית. כל חלון = Subagent נפרד שעובד על המשימה שלו.

הדמיה מעשית: פתח 3 חלונות טרמינל, כולם מכוונים לתיקיית הפרויקט, כולם מריצים claude. תן לכל אחד הוראות שונות. הם עובדים במקביל — כל אחד על קבצים שונים. בסוף ה-Git מאחד את כל השינויים.

דוגמת שימוש ב-Subagents 
הפעל 3 subagents במקביל:
1. Agent א: בדוק security vulnerabilities ב-/src/api/
2. Agent ב: כתוב unit tests לכל function ב-/src/utils/
3. Agent ג: עשה performance audit ל-/src/components/

כל agent יכתוב ממצאים לקובץ נפרד.

מה זה MCP? קיצור של Model Context Protocol — פרוטוקול (שפה משותפת) שמאפשר לClaude "להתחבר" לכלים חיצוניים ולשלוט בהם. בלי MCP, Claude רק כותב קוד. עם MCP, Claude יכול לפתוח דפדפן אמיתי, לבצע שאילתות בבסיס הנתונים, לשלוח הודעות ב-Slack — הכל ישירות.

כל MCP Server = חיבור לכלי אחד. מגדירים אותם פעם אחת ב-settings.json ואז Claude יכול להשתמש בהם בכל שיחה.

🎭
Playwright
אוטומציה של דפדפן, E2E tests, screenshot ודיבוג UI
🐘
PostgreSQL
שאילתות ישירות ל-DB, migrations, ניתוח נתונים
💬
Slack
שליחת הודעות, קריאת ערוצים, עדכון סטטוס
🎨
Figma
קריאת עיצובים, הפקת קוד מ-mockups
🐙
GitHub
PRs, issues, code reviews, workflows
🔍
Brave Search
חיפוש באינטרנט, תיעוד עדכני, Stack Overflow

      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_URL": "postgresql://user:pass@localhost/mydb"
      }
    },
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "your-token-here"
      }
    }
  }
}

פרק 12

Prompts לדוגמה שעובדים

אלו Prompts מוכנים שתוכל להעתיק ולהשתמש בהם ישירות. כל אחד מותאם לשלב שונה בworkflow.

🔍 Prompt מחקר
Deeply investigate the codebase to understand [TOPIC]. Go through every relevant file in great detail. Understand the architecture, data flow, and edge cases. Don't write any code yet. Write all your findings to research.md with clear sections.
📋 Prompt תכנון

⚡ Prompt ביצוע
The plan in plan.md has been reviewed and approved. Now execute it step by step. After each major step, confirm it works before moving on. If you encounter anything unexpected, stop and tell me before continuing.
🐛 Prompt Debug
[ PASTE ERROR + STACK TRACE ] Deeply investigate the root cause of this error. Don't fix symptoms — find and fix the actual cause. Read every relevant file. Explain what's going wrong and why, then fix it.
♻️ Prompt Refactor
Knowing everything you know about this codebase now, what is the most elegant and maintainable way to implement [FEATURE]? Scrap the current approach if needed. Think deeply, then write the plan to refactor-plan.md before touching any code.
👀 Prompt Code Review
Review the changes in [FILE/PR] like a senior engineer. Be thorough and critical. Check for: correctness, edge cases, security issues, performance problems, test coverage, and code clarity. Give specific, actionable feedback with line references.

Supabase — מדריך מהיר

⚠️
אף פעם לא להשתמש ב-schema ה-public
תמיד צור schema ייעודי לפרויקט. public חשוף לכולם כברירת מחדל.
Custom Schema Setup
-- צור schema ייעודי
CREATE SCHEMA IF NOT EXISTS app;
-- Dashboard → Settings → Integrations → Data API → Exposed schemas → הוסף "app"
TypeScript client עם custom schema
const supabase = createClient(url, key, {
  db: { schema: 'app' }
});
Table Template — כל טבלה חייבת את השדות האלו
CREATE TABLE app.customers (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL,
  phone       TEXT,
  email       TEXT,
  status_id   UUID REFERENCES app.statuses(id),
  -- Audit fields (חובה בכל טבלה)
  created_at  TIMESTAMPTZ DEFAULT now(),
  created_by  UUID REFERENCES auth.users(id),
  updated_at  TIMESTAMPTZ DEFAULT now(),
  updated_by  UUID REFERENCES auth.users(id),
  deleted_at  TIMESTAMPTZ  -- soft delete
);

GRANT ALL ON app.customers TO authenticated;

-- Triggers אוטומטיים
CREATE TRIGGER set_updated_at
  BEFORE UPDATE ON app.customers
  FOR EACH ROW EXECUTE FUNCTION app.set_updated_at();

CREATE TRIGGER set_user_fields
  BEFORE INSERT OR UPDATE ON app.customers
  FOR EACH ROW EXECUTE FUNCTION app.set_user_fields();
RLS — Dynamic Permission System
-- פונקציה אחת שמנהלת הכל דרך טבלת permissions
CREATE FUNCTION app.has_table_permission(
  user_id   UUID,
  p_table   TEXT,
  p_action  TEXT,
  row_owner UUID DEFAULT NULL
) RETURNS BOOLEAN AS $$ ... $$ LANGUAGE plpgsql SECURITY DEFINER;

-- הפעל RLS על הטבלה
ALTER TABLE app.customers ENABLE ROW LEVEL SECURITY;

-- Policy אחת לכל הפעולות
CREATE POLICY "rls_customers" ON app.customers
  USING (app.has_table_permission(auth.uid(), 'customers', 'read', created_by))
  WITH CHECK (app.has_table_permission(auth.uid(), 'customers', 'create'));

-- Views — תמיד security_invoker
CREATE VIEW app.dashboard_summary
  WITH (security_invoker = true) AS
  SELECT ... FROM app.customers WHERE deleted_at IS NULL;
GRANT SELECT ON app.dashboard_summary TO authenticated;

Scope ערכים לטבלת permissions:

  • org — גישה מלאה לכל הרשומות
  • own — רק רשומות אישיות
  • none — חסום
